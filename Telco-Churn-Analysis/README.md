📊 Telco Customer Churn Analysis

1. 프로젝트 개요
    본 프로젝트는 통신사 고객 데이터를 활용하여
    고객 이탈(Churn) 패턴을 분석하고, 이탈 가능 고객을 예측하는 모델을 구축하는 것을 목표로 함.
    데이터는 IBM Telco Customer Churn Dataset을 활용하였음.
2. 데이터 설명
    고객 수: 약 7,000명
    목표 변수(Target): Churn Value
    0 = 유지
    1 = 이탈
    주요 변수: 계약 유형 (Contract),
            가입 기간 (Tenure Months),
            월 요금 (Monthly Charges),
            결제 방식 (Payment Method),
            고객 특성 정보
3. 탐색적 데이터 분석 (EDA)
    3-1. 전체 이탈률
        전체 고객의 약 **26.5%**가 이탈
        약 4명 중 1명이 서비스를 해지하는 수준
    3-2. 계약 유형별 이탈률
        계약 유형	이탈률:
            Month-to-month	42.7%
            One year	11.2%
            Two year	2.8%
        해석:
            단기 계약 고객의 이탈 위험이 매우 높음.
            장기 계약은 강력한 이탈 방어 요인.
3-3. 가입 기간별 이탈률
가입 기간	이탈률
0~12개월	47.6%
12~24개월	28.7%
24~48개월	20.3%
48개월 이상	9.5%
해석:
가입 초기 고객에서 이탈이 집중 발생
장기 고객은 높은 유지율을 보임
3-4. 고위험 세그먼트 정의
가장 위험한 고객군:
Month-to-month 계약 & 가입 12개월 미만
→ 이탈률 약 51%
이 세그먼트를 핵심 관리 대상으로 정의하였다.
4. 모델링
    4-1. 모델 구성
        Train/Test Split (8:2)
        One-Hot Encoding 적용함.
        RandomForest Classifier 사용함.
    4-2. 기본 모델 성능
        지표	값
        Accuracy	0.79
        Churn Precision	0.64
        Churn Recall	0.52
        Churn F1-score	0.57
        해석:
            전체 정확도는 79% 수준
            실제 이탈 고객의 약 52%를 사전에 탐지 가능.
            이탈 탐지 성능 개선 여지 존재함.
    4-3. 클래스 불균형 개선 실험
        이탈 고객 비율이 약 27%로 불균형 데이터 특성이 존재함.
        이를 완화하기 위해 class_weight='balanced' 옵션을 적용함.
        모델	Accuracy	Recall (Churn)	F1-score
        기본 모델	0.79	0.52	0.57
        Balanced 모델	0.79	0.51	0.57
        해석:
            가중치 조정만으로는 성능 개선 효과가 크지 않았음.
            RandomForest는 불균형 데이터에 비교적 강건한 특성을 보임.
            향후 하이퍼파라미터 튜닝 또는 다른 모델 적용 필요.
5. 비즈니스 인사이트
    단기 계약 고객을 장기 계약으로 전환 유도 필요함.
    가입 초기 3~6개월 집중 관리 프로그램 설계 필요함.
    고위험 고객군 대상 맞춤형 리텐션 전략 수립 가능.
6. 프로젝트를 통해 배운 점
    데이터 전처리의 중요성 (결측치, 타입 변환)
    이탈 문제에서 Recall 지표의 중요성 이해함.
    단순 정확도보다 비즈니스 관점 해석이 중요함을 학습함.
